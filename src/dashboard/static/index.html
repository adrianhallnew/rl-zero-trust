OpenZiti Zero-Trust Layer

Identity-Based Access

Every component (Ryu, Mininet, RL Agent) enrolls with a unique X.509 identity. No IP-based trust — all connections authenticated via mutual TLS certificates issued by the OpenZiti controller.

Dark Services

Services are invisible to unauthorized identities. The RL agent has no open inbound ports — it initiates all connections outward. An nmap scan of 172.20.0.40 returns zero open ports, demonstrating the dark service model.

mTLS Mesh

All inter-component communication is encrypted end-to-end with mutual TLS. Session tokens are never stored — each connection is independently authenticated against the OpenZiti PKI. HTTP fallback available when ZT overlay is disabled.
